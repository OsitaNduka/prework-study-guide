HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • HTML elements composed of opening tag and closing tag
  • Elements that the user will see and interact with must be placed in the body
  • Child elements are nested on the body element and the elements that holds the child element is the parent element
  • The body element is an example of a semantic elements

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS defines colours, aesthetics, visual layout, organization, and a wide range of style options
  • CSS are used to style an element directly in the HTML or XML file
  • CSS is created to allow the seperation of content and presentation of a webpage
  • CSS outlines how elements should be interpreted on screen, paper, in speech or on other media

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add: add a file as it looks now to our next commit (stage)
  • git reset: unstage a file while retaining the changes in working directory
  • git diff: diffing(takes two input sets of data and outputs the changes between them) (diff) of what is changed but not staged
  • git diff ---staged: diff of what is staged but not yet committed
  • git commit -m "descrptive message" commit our staged contents as a new snapshot
  • git branch: list our branches. * will appear next to the currently active branch
  • git branch [branch name]: create a new branch at the current commit
  • git checkout: switch to another branch and check it out into our working directory
  • git merge: [branch]: merge the specified branch's history into the current one
  • git log: show all commits history for the current active branch
  • git clone: reterieve an entire repository from a hosted location via URL
  • git init: initialise an existing directory as a Git repository
  • git log branchB. .branchA: show the commits on branchA that are not in branchB
  • git log --follow [file]: show the commits that changed file, even across renames
  • git diff branchB...branchA: show the diff of what is in branchA that is not in branchB
  • git show [SHA]: show any object in Git in human-readable format
  • git rm [file]: delete the file from project and stage the removal for commit
  • git mv [existing-path] [new]=path]: change an existing file path and stage the move
  • git log --stat -M: show all commit logs withindication of any paths that moved
  • git remote add [alias] [url]: add a git URL as an alias
  • git fetch [alias]: fetch down all the branches from that Git remote
  • git merge [alias]/[branch]: merge a remote branch into our current branch to bring it up to date
  • git push [alias] [branch]: transmit local branch commits to the remote repository branch
  • git pull: fetch and merge any commits from the tracking remote branch
  • git rebase [branch]: apply any commits of current branch ahead of specified one
  • git reset --hard [commit]: clear staging area, rewrite working tree from specified commit
  • git stash: save modified and staged changes
  • git stash list: list stack-order of stashed file changes
  • git stash pop: write working from top of stash stack
  • git stash drop: discard the changes from top of stash stack

Javascript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Conditional statements such as "if" statements allows us to interupt control flow
  • Javascript is a high level dynamic interpreted programming language: excuted directly without having to run through a compiler
  • Javascript: encrypted and interpreted; it adds interactivety to a website
  • Javascript conforms to the ECMAScript specification
  • Javascript is multi-paradigm: able to write code in many diferent ways (Paradigm) (functional code etc)
  • Javascript runs on the client browser as well as on the server (Node.js)
  • Javascript is the programming languge of the browser: Application Programming Interfaces(APIs) with frameworks such as React
  • Javascript is used in mobile development (React Native, NativeScript, Ionic)
  • Javascript is used in building very fast server side and full stack applications
  • Javascript is used in desktop application development (Electron. js)
